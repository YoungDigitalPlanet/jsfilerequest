JsFileRequest

A GWT library for fetching files that can work in two modes:

  • Standard Mode - fetching files via HTTP GET requests
  • JavaScript Inject Mode - fetching files via script injection technique; appending <script> tag in <head> with the requested file, the subject file should be prepared
Usable when the web application is to be run locally (from file:///). IE, Chrome and Opera won't allow XmlHttpRequest for loading local files.

Preparing content files

The value of the jsFileRequestSuffix variable should be appended to all file names. Please remember to escape quotation marks and add next line markers.

The files should be prepared as shown in the table below:

Original file contents Prepared file contents (see underlined fragments)
script_00001_001.emp.xml script_00001_001.emp.xml.js
Lorem 'ipsum' dolor sit amet.
Curabitur viverra dui et tortor volutpat at dignissim metus adipiscing. 		window.parent.jsFileRequestReceiveFile(
'Lorem \'ipsum\' dolor sit amet.\n\
Curabitur viverra dui et tortor volutpat at dignissim metus adipiscing.
');

The references to the files should not change. It means that if script_00001.emt.xml in its contents refers to script_00001_001.emp.xml it must NOT be changed to script_00001_001.emp.xml.js. The JsFileRequest library will add the suffix automatically.

Typical usage

Put the below code into your HTML host page:

<script type="text/javascript" language="javascript">
var jsFileRequestSuffix = ".js"; // the suffix added to the names' of all files that will be loaded
var jsFileRequestPerformer = "jsinject"; // "jsinject" or "default" (default value)
var jsFileRequestEnableLogging = true; // set true to enable logging in the console, false or do not define the variable otherwise
</script>
